## 项目简介

PyCallTrace 是一个 Python 函数调用追踪与分析工具，主要功能包括：

✅ **自动追踪** - 基于 `sys.setprofile()` 全局钩子，自动拦截所有函数调用  
✅ **线程支持** - 支持多线程程序的调用追踪  
✅ **可视化分析** - 纯前端可视化工具，树状展示调用关系  
✅ **源码映射** - 可直接在浏览器中查看调用栈对应的源代码  
✅ **零依赖前端** - 日志分析器无需任何框架或依赖，纯 HTML/CSS/JS 实现


## 快速开始

### 第一步：安装依赖

打开终端或命令行，执行以下命令安装必要的依赖：

```bash
pip install loguru
```

> **说明**：本项目仅依赖 `loguru` 库，用于格式化日志输出。

---

### 第二步：生成日志文件

在项目根目录下执行：

```bash
python main.py
```

**执行效果**：

控制台会输出彩色格式的追踪日志，包含：
- 线程 ID（tid）
- 函数进入（enter fun）
- 函数退出（exit fun）
- 普通日志消息

**示例输出**：
```
tid:12345|enter fun:func_a@C:\hh\git\PyCallTrace\function_calls.py:23
tid:12345|enter fun:func_b@C:\hh\git\PyCallTrace\function_calls.py:45
tid:12345|exit fun:func_b@C:\hh\git\PyCallTrace\function_calls.py:45
tid:12345|exit fun:func_a@C:\hh\git\PyCallTrace\function_calls.py:23
```

> **提示**：日志输出到控制台，如需保存到文件，可使用重定向：
> ```bash
> python main.py > 1.log
> ```

---

### 第三步：使用日志分析器

有两种方式打开日志分析器：

#### 方式一：直接打开

在文件浏览器中双击 `log_analyzer.html`，或右键选择浏览器打开。

#### 方式二：使用本地服务器（推荐）

```bash
python serve.py
```

然后在浏览器中访问：`http://localhost:8080/log_analyzer.html`

> **为什么推荐服务器方式？**  
> 某些浏览器因安全限制，可能无法正确加载本地文件。

## 详细使用说明

### 一、使用追踪器生成日志

#### 1. 运行追踪器

```bash
python main.py
```

程序会自动执行 `function_calls.py` 中的业务函数，并输出追踪日志。

#### 2. 日志格式说明

每条日志遵循统一格式：

```
tid:{线程ID}|{事件类型} fun:{函数名}@{文件路径}:{行号}
```

**事件类型**：

| 类型 | 示例 | 说明 |
|------|------|------|
| `enter fun` | `tid:12345|enter fun:func_a@C:\path\file.py:23` | 函数被调用 |
| `exit fun` | `tid:12345|exit fun:func_a@C:\path\file.py:23` | 函数正常返回 |
| `exception fun` | `tid:12345|exception fun:func_a@C:\path\file.py:23` | 函数抛出异常 |
| 普通消息 | `tid:12345|程序开始执行` | 开发者自定义的日志 |

#### 3. 重定向日志到文件

```bash
python main.py > 1.log
```

---

### 二、使用日志分析器查看日志

#### 1. 打开日志分析器

在浏览器中打开 `log_analyzer.html`，界面如下：

```
┌─────────────────────────────────────────────────────┐
│  📊 日志分析工具    [📂 打开日志文件] [📁 映射目录]  │
├──────────────┬──────────────────────────────────────┤
│              │                                      │
│  线程列表     │         日志内容（调用树）            │
│              │                                      │
│  ◉ tid:12345 │  ▶ func_a (function_calls.py:23)     │
│    tid:12346 │    ▶ func_b (function_calls.py:45)   │
│    tid:12347 │       普通日志消息                    │
│              │                                      │
│  共 3 个线程  │                                      │
└──────────────┴──────────────────────────────────────┘
```

#### 2. 加载日志文件

**步骤**：

1. 点击顶部工具栏的 **📂 打开日志文件** 按钮
2. 在弹出的文件选择器中，选择 `.log` 文件
3. 文件加载完成后，左侧面板会显示所有发现的线程

**效果**：

- **左侧面板**：显示所有线程 ID 列表
- **右侧面板**：提示"选择线程查看日志内容"

#### 3. 选择线程查看调用树

**步骤**：

1. 在左侧线程列表中，点击要查看的线程（如 `tid:12345`）
2. 选中后，该线程会高亮显示（蓝色背景）
3. 右侧面板会显示该线程的完整调用树

**调用树说明**：

```
▶ func_main (function_calls.py:400)
  ├─ ▶ func_a (function_calls.py:23)
  │   ├─ ▶ func_b (function_calls.py:45)
  │   │   └─  达到最大深度,返回
  │   └─  随机选择起始函数: func_b
  └─ ▶ func_c (function_calls.py:67)
      └─ ▶ func_d (function_calls.py:89)
```

**交互操作**：

| 操作 | 效果 |
|------|------|
| 点击 `▶` 箭头 | 展开/折叠子调用 |
| 点击函数名 | 无（仅显示） |
| 点击文件名 | 打开源代码查看器 |
| 点击行号 | 无（仅显示） |

### 三、映射源代码目录

为了在分析器中直接查看源代码，需要先映射包含 `.py` 文件的目录。

#### 1. 映射目录

**步骤**：

1. 点击顶部工具栏的 **📁 映射目录** 按钮
2. 在弹出的选择器中，选择包含 Python 源文件的目录（如 `PyCallTrace` 根目录）
3. 确认选择后，工具会扫描该目录下的所有 `.py` 文件

**效果**：

- 按钮变为激活状态（绿色高亮）
- 弹出提示框显示统计信息：
  ```
  ✓ 成功加载 2 个 .py 文件
  
  ℹ 目录中共 10 个文件，其中 8 个非 .py 文件已忽略
  ```

#### 2. 映射目录的限制

| 限制项 | 说明 |
|--------|------|
| 最大文件数 | 1000 个 `.py` 文件 |
| 单文件大小 | 5 MB |
| 文件类型 | 仅支持 `.py` 文件 |

> **提示**：如果目录中有大量非 `.py` 文件，工具会自动忽略它们。

### 四、查看源代码

#### 1. 从调用树打开源代码

**步骤**：

1. 在调用树中，找到要查看的函数节点
2. 点击节点中的 **文件名**（黄色加粗文本，如 `function_calls.py`）
3. 源代码查看器会出现在界面底部

**效果**：

```

│  47          return                         │

```

#### 2. 从日志消息打开源代码

如果日志消息中包含文件信息（格式：`文件名:行号|消息内容`），可以直接点击文件名查看源码。

**示例日志**：

在调用树中，`function_calls.py:45|` 会显示为可点击的链接。

|------|------|

### 调用树可视化

## 常见问题

### Q1: 运行 `python main.py` 时提示找不到 `loguru` 模块

**解决方法**：

```bash
pip install loguru
```

如果仍然报错，检查 Python 环境：

```bash
python --version
pip list | grep loguru
```

---

### Q2: 浏览器打开 `log_analyzer.html` 后无法加载日志文件

**原因**：某些浏览器（如 Chrome）因安全限制，不允许直接读取本地文件。

**解决方法**：

1. 使用本地服务器：
   ```bash
   python serve.py
   ```
   然后访问 `http://localhost:8080/log_analyzer.html`

2. 或者使用 Firefox 浏览器（对本地文件的限制较宽松）

---

### Q3: 日志分析器中看不到任何线程

**原因**：日志文件格式不正确，或未包含 `tid:` 前缀。

**检查步骤**：

1. 确认日志文件是由 `main.py` 生成的
2. 打开日志文件，检查是否有类似以下格式的行：
   ```
   tid:12345|enter fun:func_a@C:\path\file.py:23
   ```

---

### Q4: 点击文件名后提示"无法读取文件"

**原因**：映射目录中未包含对应的 `.py` 文件。

**解决方法**：

1. 点击 **📁 映射目录** 按钮
2. 选择包含该 `.py` 文件的目录
3. 如果仍然找不到，点击 **选择源文件** 手动定位文件

---

### Q5: 调用树显示不完整或有错误

**原因**：日志文件可能不完整，或存在格式错误。

**解决方法**：

1. 重新运行 `python main.py` 生成新的日志文件
2. 检查日志文件中是否有不完整的行
3. 确保日志文件编码为 UTF-8

---

### Q6: 如何复现特定的调用链？

`function_calls.py` 使用随机选择，每次运行的调用链都不同。

**复现方法**：

在 `function_calls.py` 的 `main()` 函数中，取消注释随机种子：

```python
def main():
    # 取消注释下面的行，可以复现结果
    random.seed(42)
    
    logger.debug("=" * 80)
    logger.debug("程序开始执行 - 使用loguru日志库")
    # ...
```

设置相同的随机种子，每次运行都会生成相同的调用链。

## 附录

### A. 日志格式详解

**完整格式**：

```
tid:{线程ID}|{事件类型} fun:{函数名}@{文件路径}:{行号}
```

**示例**：

```
tid:20816|enter fun:func_t@C:\hh\svn\PyCallTrace\function_calls.py:419
```

**字段说明**：

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `tid` | 线程 ID（数字） | `20816` |
| `事件类型` | `enter fun` / `exit fun` / `exception fun` | `enter fun` |
| `函数名` | 被调用的函数名称 | `func_t` |
| `文件路径` | 源文件的完整路径 | `C:\hh\svn\PyCallTrace\function_calls.py` |
| `行号` | 函数定义的起始行号 | `419` |

---

### B. 业务函数调用关系图

`function_calls.py` 包含 20 个函数（`func_a` ~ `func_t`），每个函数随机调用其他函数：

```
func_a → func_b / func_c / func_d
func_b → func_a / func_e / func_f
func_c → func_d / func_g / func_h
func_d → func_a / func_e / func_i
func_e → func_b / func_f / func_j
func_f → func_c / func_g / func_k
func_g → func_a / func_d / func_l
func_h → func_b / func_e / func_m
func_i → func_c / func_f / func_n
func_j → func_d / func_g / func_o
func_k → func_a / func_h / func_p
func_l → func_b / func_i / func_q
func_m → func_c / func_j / func_r
func_n → func_d / func_k / func_s
func_o → func_e / func_l / func_t
func_p → func_f / func_m / func_a
func_q → func_g / func_n / func_b
func_r → func_h / func_o / func_c
func_s → func_i / func_p / func_d
func_t → func_j / func_q / func_e
```

**调用深度保护**：

- 最大调用深度：`20` 层
- 超过深度后，函数会打印 `达到最大深度,返回` 并直接返回
